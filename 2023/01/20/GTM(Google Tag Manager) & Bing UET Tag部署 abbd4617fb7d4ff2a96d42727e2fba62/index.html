GTM(Google Tag Manager) & Bing UET Tag部署

Last updated on December 22, 2023 pm

GTM(Google Tag Manager) & Bing UET Tag部署

mireia

基础代码

  1. 基础代码：后台创建基础代码，复制代码ID

  2. 选择容器：在Google tag manager选择容器

    一个容器ID对应一个网站。容器ID应与网站中安装的ID保持一致

    点击新建Tag，进入Tag Configuration

  3. 创建代码：选择Microsoft 广告

  4. 填写信息：粘贴bing UET tag ID，补充剩余信息，（UETQ ID，只有在一个网站中存在多个UET代码时需要分别命名，如Uetq1，uetq2进行区分，代码唯一时，可以忽略）

  5. 设置基础代码Trigger，选择All page

    Trigger指代码触发网页，
    UET基础代码应用全局，我们设置为 All page。设置付费代码或者事件代码等应用于特定页面的代码时，trigger应设置为some page

  6. 保存并命名代码，检查代码是否生效

统计金额代码

  1. 根据追踪事件创建conversion goal，此处以付费事件为例，定位方式是结算页URL的Thank_you字符。（可以根据网站选定其他字符进行定位，但需要确保该字符不会在网站其他页面被触发，避免conversion数据错误计入）

  2. 在GTM设置Trigger，设置为进入thank you界面的用户

  3. 设置变量：为需要统计的事件设置变量，这里以统计金额和货币单位为例

    创建变量： 根据站点本身对于统计金额和货币单位的命名方式填写变量名称，变量类型选择：Data Layer Variable

  4. 设置付费页Trigger，类型选择page type，

    根据在conversion goal中的设置，对页面进行定位，这里是设置为page URL，包含 ，thank_you

  5. 设置付费 Tag

    从**“Event type**”下拉列表中选择variable revenue for destination URL

    关联两个取值变量

    在下方关联之前设置的trigger, 完成后保存并发布

代码检查

**安装插件：UET Tag Helper，**打开待检测网站，打开插件
