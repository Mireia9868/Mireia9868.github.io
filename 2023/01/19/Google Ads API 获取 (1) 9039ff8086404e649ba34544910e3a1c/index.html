在 Google Ads 网页界面上完成 Google Ads API 注册，即可自动获得测试帐号访问权限级别。

第 2 步：申请访问 Google Ads API

登录您的经理帐号，然后前往工具和设置 > 设置 > API 中心。系统仅针对 Google Ads 经理帐号显示 API 中心选项。

然后，填写 API 访问权限表单并接受条款及条件。

  • 请确保您的信息正确无误，并且贵公司的网站网址也能正常运行。如果该网站未上线，Google 可能无法处理您的应用并拒绝它。

  • 确保您提供的 API 联系电子邮件地址是您定期查看的收件箱。在审核期间，Google 的 API 合规团队可能会与此电子邮件地址联系以澄清说明。如果无法访问，Google 可能不会继续处理您的应用。

    您可以在 API 中心修改 API 联系电子邮件地址。即使在申请流程之后，也请及时更新这些信息，以便 Google 可以向您发送重要的服务公告。

完成申请流程后，开发者令牌会以待审批状态显示在您的 API 中心。您的开发者令牌现在具有测试访问权限级别，您可以开始对测试帐号发出 API 调用。

获取开发者令牌后，请记下其值，因为您在进行首次 API 调用时需要用到它。

创建一个 Google API 控制台项目

https://console.developers.google.com/project?hl=zh-cn

  • 前往 Google API 控制台
  • 点击创建项目
  • 输入名称或接受生成的建议
  • 确认或修改其余所有字段
  • 点击创建

  1. 打开 “同意屏幕”页面，然后在页面顶部的下拉菜单中选择您的项目。

  2. 选择用户类型，然后点击创建。

  3. 填写同意屏幕表单。如果需要，您以后可以进行修改。 完成后，点击保存并继续。

  4. 在下一页上，点击添加或移除范围。

  5. 向下滚动，然后在手动添加范围下的字段中输入以下内容：

    https://www.googleapis.com/auth/adwords

  6. 点击添加到表格。

  7. 添加项目可能需要的任何其他范围，然后点击更新。

  8. 点击保存并继续。

  9. 如果您不打算立即发布项目，请添加您可能用于测试的每个用户帐号的电子邮件地址，然后点击保存并继续。

  10. 查看摘要页面，然后点击返回信息中心。

在项目中启用 Google Ads API

To enable the Google Ads API in your project, follow these steps:要在项目中启用 Google Ads API，请按以下步骤操作：

  1. Open the API Library in the Google API Console. If prompted, select your project or create a new one. The API Library lists all available APIs, grouped by product family and popularity.在 Google API 控制台中打开 API 库。如果出现提示，请选择您的项目或创建一个新项目。API 库列出了所有可用的 API，并按产品系列和受欢迎程度分组。
  2. Use search to find the Google Ads API if it isn’t visible in the list.如果列表中未显示 Google Ads API，请使用搜索来查找该 API。
  3. Select the Google Ads API, then click the Enable button.选择 Google Ads API，然后点击启用按钮。

准备客户帐号

特征集的选择

选择用于调用 API 的帐号时，还需要考虑要与 API 搭配使用的功能列表。即使您选择使用生产帐号，也有一些特定于帐号的功能，例如：

  • 要使用结算功能，您需要一个采用按月账单结算的帐号。
  • 部分新功能可能仅限列入许可名单的帐号列表使用。

在最终确定进行 API 调用的帐号之前，请查看特定功能的文档，以确保该功能没有其他限制。

用户访问权限、登录客户 ID 和客户 ID

要向 Google Ads 客户进行 API 调用，您需要直接访问该客户或通过关联的经理帐号进行访问。您还需要根据特定的 API 调用拥有适当的权限。

登录 Google Ads 时，您可能会有多个客户帐号可供选择。您选择的帐号会建立登录客户 ID（某些 API 调用的必需标头）和您的访问权限级别。

确定要向其发出 API 调用的 Google Ads 客户后，请注意以下详细信息：

  • 您计划向其发出 API 调用的客户帐号的 10 位数客户 ID。这有时也称为操作客户。
  • 您所登录的客户帐号的电子邮件地址。需要此令牌才能生成 OAuth 2.0 刷新令牌。
  • 您所登录客户帐号的 10 位数登录客户 ID。 如果您已登录正在运行的客户，则登录客户 ID 与客户 ID 相同

邀请用户后，注意前往邮箱激活邀请以确保可以正常进入

下载工具和客户端库

bookmark_border

下载以下工具以完成本教程的其余步骤。

OAuth2L

Google Ads API 使用 OAuth 2.0 作为授权机制。本教程使用 oauth2l，这是一种用于处理 Google OAuth 2.0 的命令行工具，用于获取和输出访问令牌和刷新令牌。您可以从 GitHub 代码库下载并安装预编译的二进制文件。

客户端库或 HTTP 客户端

根据所需的 API 调用方式，您可以选择下载客户端库或 HTTP 客户端。
